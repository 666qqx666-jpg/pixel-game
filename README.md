# 像素闯关问答（Pixel Quiz）

基于 **React + Vite** 的前端闯关游戏：从 **Google 表格** 随机抽题，经 **Google Apps Script（GAS）** 计分并写回「回答」表。界面为像素街机风格，关主头像使用 [DiceBear](https://www.dicebear.com/) 的 `pixel-art` 样式。

---

## 本地运行

### 环境要求

- Node.js 18+（建议 LTS）
- 现代浏览器（Chrome / Edge / Firefox / Safari）

### 安装与启动

```bash
git clone <你的仓库地址> pixel-game
cd pixel-game
npm install
```

复制环境变量模板并编辑：

```bash
cp .env.example .env
```

在 `.env` 中填写（**不要**把 `.env` 提交到公开仓库）：

| 变量 | 说明 |
|------|------|
| `GOOGLE_APP_SCRIPT_URL` | GAS 部署后的 **Web 应用网址**（以 `/exec` 结尾的部署链接） |
| `PASS_THRESHOLD` | 通关需要答对的题目数量（整数） |
| `QUESTION_COUNT` | 每一局随机抽取的题目数量（整数，不能超过「题目」表中有效行数） |

启动开发服务器：

```bash
npm run dev
```

生产构建：

```bash
npm run build
```

构建结果在 `dist/`，可用任意静态服务器托管，也可用 `npm run preview` 本地预览。

### GitHub Pages（避免白屏、JS 404）

**不要**只开启 Pages 并指向 `main` 分支**根目录**：仓库根目录是源码，`index.html` 会请求 `/src/main.tsx`，线上没有 Vite，必然 404、白屏。

推荐两种方式（二选一）：

1. **GitHub Actions（本仓库已含工作流）**  
   - 仓库 **Settings → Pages → Build and deployment → Source** 选 **GitHub Actions**。  
   - **Settings → Secrets and variables → Actions** 新建 **Repository secret**：`GOOGLE_APP_SCRIPT_URL`（填你的 GAS `/exec` 地址）。  
   - 推送 `main` 后会自动 `npm ci` → `npm run build`（`BASE_PATH` 自动等于**仓库名**）→ 部署 `dist/`。  
   - 访问：`https://<用户名>.github.io/<仓库名>/`（例如 [pixel-game 仓库](https://github.com/666qqx666-jpg/pixel-game) 对应 `https://666qqx666-jpg.github.io/pixel-game/`）。

2. **本地构建后手动上传**  
   - `.env` 中设 `BASE_PATH=你的仓库名`，执行 `npm run build`，把 **`dist/` 内全部文件** 传到 `gh-pages` 分支**根目录**（或任意静态托管的根目录）。

`index.js` / `content_script.js` / Chrome Built-In AI 等多为**浏览器扩展**日志，与项目无关，可忽略。

## Google 表格：创建与结构

所有数据在同一份 **Google 表格** 中；脚本与表格绑定（从该表格打开 Apps Script）。

### 1. 新建表格

1. 打开 [Google 表格](https://sheets.google.com)，点击 **空白** 新建表格。
2. 建议将文件命名为例如：`像素闯关题库`。
3. 底部标签页需要 **两个工作表**，名称必须 **与下表完全一致**（脚本里写死了表名）：

| 工作表名称 | 用途 |
|------------|------|
| `题目` | 存放选择题题干与答案键 |
| `回答` | GAS 写入玩家闯关统计 |

- 默认第一个表可重命名为 `题目`：**右键标签 → 重命名**。
- 点击 **+** 添加第二个表，重命名为 `回答`。

### 2.「题目」表：表头（第一行）

第一行 **逐字** 使用下列列名（区分大小写：`A` `B` `C` `D` 为大写英文字母）：

`题号` | `题目` | `A` | `B` | `C` | `D` | `解答`

- **题号**：建议用 `1`、`2`、`3` … 或 `Q001` 等字符串，只要每题唯一即可（提交成绩时靠题号对齐答案）。
- **题目**：题干文字。
- **A～D**：四个选项的文案。
- **解答**：正确选项，填 **单个字母** `A`、`B`、`C` 或 `D`（脚本会转成大写比对）.

从第二行开始，每行一题。文末 **「附录：测试题目」** 中提供了 10 道「生成式 AI 基础知识」样题，可直接复制到表中。

### 3.「回答」表：表头（第一行）

第一行 **逐字** 为：

`ID` | `闯关次数` | `总分` | `最高分` | `第一次通关分数` | `花了几次通关` | `最近游玩时间`

- 第二行可以先留空，或作为说明行删除；**脚本只认第一行表头**，从第二行开始找 `ID` 匹配玩家。
- 首次有玩家提交后，GAS 会 **追加或更新** 行：`闯关次数` 每次完整一局 +1；`总分` 为 **本局答对题数**；`最高分` 为历史最高；首次通关时写入 `第一次通关分数` 与 `花了几次通关`，之后通关不再覆盖这两项，只更新次数与分数统计；`最近游玩时间` 由脚本写入。

## Google Apps Script：绑定与部署

本仓库脚本路径：`google-apps-script/Code.gs`。请把其中 **全部代码** 拷贝到与你表格绑定的项目中。

### 1. 打开脚本编辑器

1. 打开 **已建好「题目」「回答」两个工作表** 的那份 Google 表格。
2. 菜单：**扩展程序** → **Apps Script**（界面语言不同可能是「扩展功能」等）。
3. 若左侧默认有 `代码.gs` 或 `Code.gs`，可 **全选删除** 原内容，再粘贴本项目的 `Code.gs` 全文。

### 2. 保存项目

- `Ctrl/Cmd + S` 保存，项目名称可改为例如 `PixelQuizBackend`。

### 3. 授权（首次运行必做）

1. 点击 **运行**，选任意函数（若没有可暂时不用跑；部署后再用前端测也行）。
2. 首次会提示 **授权**：选择 Google 账号 → **高级** → **前往…（不安全提示时）** → **允许**。
3. 需同意脚本访问 **当前电子表格**。

若组织策略禁止“任何人”访问 Web 应用，需由管理员调整或改用登录用户部署（见下文 troubleshooting）。

### 4. 部署为 Web 应用

1. 右上角 **部署** → **新建部署**。
2. 点击「选择类型」齿轮旁，选 **Web 应用**。
3. 建议设置：
   - **说明**：例如 `v1 答题 API`。
   - **执行身份**：**我**（你的账号有表格编辑权限）。
   - **具有访问权限的用户**：
     - 面向匿名玩家：**任何人**（Anyone）——前端才能用 `fetch` 跨域调用。
     - 若只能选“仅限组织内”，则仅同组织账号能玩，或个人测试时用登录态。
4. 点击 **部署**，复制 **Web 应用网址**（形如 `https://script.google.com/macros/s/.../exec`）。
5. 将网址粘贴到项目根目录 `.env` 的 `GOOGLE_APP_SCRIPT_URL=` 后面 **不要** 多加引号。

### 5. 修改代码后重新部署

每次改 `Code.gs` 后：**部署** → **管理部署** → 当前部署 **修改**（铅笔图标）→ 版本选 **新版本** → **部署**。前端 `.env` 的 URL 一般 **不变**，除非新建了另一份部署。

## 联调检查清单

1. **表名**：`题目`、`回答` 无错字、无多余空格。
2. **题目表** 至少 `QUESTION_COUNT` 行有效题目（题目列非空）。
3. **解答** 列均为 `A`–`D` 之一。
4. GAS 部署为 **Web 应用**，URL 使用 **/exec** 链接。
5. `.env` 须放在 **项目根目录**（与 `vite.config.ts` 同级）。改完变量后 **必须重启** `npm run dev`（Vite 只在启动时注入）。若在上一级目录执行 npm，仍可从该根目录读取（配置已按配置文件所在目录加载 `.env`）。
6. 若浏览器控制台报 CORS 或网络错误：多为部署非「任何人」、或 URL 复制成 `/dev` 编辑链接而非用户访问的 `/exec`。
